# VGIK - Универсальная программная платформа для роутеров

## Описание проекта

**VGIK** — это гибкая, модульная и легко адаптируемая прошивка для маршрутизаторов, которая работает на широком спектре аппаратных платформ (разных чипсетов и архитектур) и предоставляет единый программный интерфейс как для производителей оборудования, так и для интернет-провайдеров.

### Целевая аудитория

- **Производители оборудования**: получают готовую базовую ОС для своих роутеров — достаточно прошить устройство, и оно готово к продаже
- **Интернет-провайдеры**: могут легко и быстро подключать устройства к своей инфраструктуре без необходимости глубокой кастомизации под каждую модель

## Ключевые возможности

### Для производителей
- ✅ Готовая прошивка "из коробки"
- ✅ Поддержка множества аппаратных платформ (MIPS, ARM, x86)
- ✅ Простое портирование под новое железо
- ✅ Автоматизированная сборка образов под разные модели
- ✅ Документация по портированию

### Для провайдеров
- ✅ Единый программный интерфейс для всех моделей
- ✅ Простое подключение устройств к сети провайдера
- ✅ Гибкая настройка под сервисы провайдера (PPPoE, VLAN, TR-069 и т.п.)
- ✅ Удалённое управление устройствами
- ✅ Поддержка протоколов управления (TR-069, NETCONF, REST API)

## Технологический стек

### Языки программирования
- **C/C++** — основной язык для низкоуровневых компонентов и работы с железом
- **Python** — для сервисов и утилит управления
- **Go** — для высокопроизводительных сетевых сервисов (по необходимости)

### Операционная среда
- **Базовая ОС**: Linux (на базе OpenWrt с возможностью форка и кастомизации)
- **Ядро**: Linux kernel с поддержкой различных архитектур

### Инфраструктура разработки
- **Система сборки**: кросс-компиляция с поддержкой множества целевых архитектур
- **CI/CD**: GitLab CI / GitHub Actions
- **Конфигурация**: JSON/YAML с валидацией схемы

### Протоколы и интерфейсы
- **TR-069** — управление устройствами провайдером
- **NETCONF** — конфигурация сетевых устройств
- **REST API** — программный интерфейс для интеграции
- **Web-интерфейс** — пользовательский интерфейс управления

## Архитектура

Платформа построена на принципах модульности и декомпозиции:

- **Ядро системы** — базовые компоненты, не зависящие от конкретного железа
- **Аппаратный слой абстракции (HAL)** — унифицированный интерфейс для работы с железом
- **Сервисный слой** — модульные сервисы (сеть, безопасность, управление)
- **API-слой** — единые интерфейсы для управления и конфигурации
- **Слой интеграции** — адаптеры для различных протоколов и систем

Подробное описание архитектуры см. в [Дизайн.md](Дизайн.md).

## Быстрый старт

### Требования
- Linux-окружение для сборки (рекомендуется Ubuntu 22.04+)
- Docker (для изолированной сборки)
- Git
- Базовые инструменты разработки (gcc, make, etc.)

### Сборка

```bash
# Клонирование репозитория
git clone <repository-url>
cd VGIK

# Настройка окружения сборки
./build/setup.sh

# Сборка для конкретной платформы
./build/build.sh --target arm64 --model example-router-v1

# Сборка образа прошивки
./build/build-firmware.sh --target arm64 --model example-router-v1
```

### Разработка

```bash
# Запуск тестов
make test

# Линтинг кода
make lint

# Форматирование кода
make format
```
